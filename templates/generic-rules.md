# Weekly Log (업무 기록)

주간 업무 로그 저장소: `{{LOG_PATH}}/weekly_log_YYYY-MM-DD.md`
(파일명 날짜 = 해당 주 월요일. 템플릿: `{{LOG_PATH}}/_TEMPLATE.md`)
이 저장소는 사용자의 여러 AI 도구가 공유한다.

## 기록 규칙
- 기록·조회 전 반드시 셸 명령(`date`, PowerShell `Get-Date` 등)으로 실제 오늘 날짜를 확인해 주차 파일을 결정한다.
  세션이 오래 유지되면 컨텍스트의 날짜는 과거(세션 시작일)일 수 있으므로 신뢰하지 않는다.
- 대화 중 아래 이벤트가 발생하면 즉시 해당 주차 로그의 `## 기록` 섹션에 `- [태그] 내용` 형식으로 append:
  설계 결정 확정, 구현/수정 완료, 배포, 중요한 디버깅 여정(원인·해결 포함), 사용자가 전달한 회의·일정 내용
- 태그는 프로젝트/모듈 경로 방식: {{PROJECT_TAG_EXAMPLES}}.
  모노리포는 `[프로젝트(모듈)]` 처럼 모듈까지 표기.
- 개발 업무가 아닌 항목(문서 작성, 회의, 리서치 등)은 성격에 맞는 태그를 임의 부여:
  `[문서]`, `[회의]`, `[리서치]`, `[보고]` 등. 기존 로그에서 쓰던 태그가 있으면 재사용.
- 해당 주차 파일이 없으면 `_TEMPLATE.md` 구조로 새로 생성하고, 직전 주차의 미완료 `[ ]` 항목을 "이번 주 목표"로 이월.
- 기존 로그는 절대 덮어쓰지 말고 append. 사소한 잡담·단순 질문은 기록하지 않는다.
- 인사이트성 내용은 `## 이번 주 배운 것`에 기록. 파일 인코딩은 UTF-8 유지.
- 기록 후 git 커밋은 하지 않는다 (사용자가 요청할 때만).

## 조회 규칙
- "이번 주 뭐 했어?" → 이번 주 월요일 날짜 파일을 읽고 태그별로 묶어 요약.
- "저번 주 그 이슈", "그 프로젝트 언제 했더라?" → `weekly_log_*.md` 전체를 키워드/태그로 검색해 주차별 시간순 제시.
- "주간 보고서 써줘" → 이번 주 로그를 프로젝트별로 묶어 완료/진행중/예정 구분한 초안 작성.

## 정리 규칙 ("이번 주 기록 정리해줘" — 사용자가 명시적으로 요청할 때만)
- 대상 주차 로그(기본: 이번 주)에서 같은 주제로 쪼개진 항목들(예: "X 검토", "X 1단계", "X 2단계")을 찾아
  주제별 **하나의 항목으로 통합**: 설계/방향 결론, 구현 완료 범위, 검증 결과, 잔여 작업만 남기고
  중간 시행착오·단계별 진척·세부 수정 과정은 제거.
- 교훈성 내용은 `## 이번 주 배운 것` 섹션으로 이동. 문서 경로·날짜 범위·진행 상태는 보존.
- 정리 결과를 사용자에게 요약 보고 (몇 개 항목이 몇 개로 정리됐는지).
