# Weekly Log (업무 기록)
  - 위치: `{{LOG_PATH}}/weekly_log_YYYY-MM-DD.md` (해당 주 월요일 날짜)
  - 대화 중 아래 이벤트가 발생하면 즉시 해당 주차 로그에 기록할 것:
    - 설계 결정 확정, 구현/수정 완료, 배포, 중요한 디버깅 여정(원인·해결 포함), 사용자가 전달한 회의·일정 내용
  - 태그는 프로젝트/모듈 경로 방식: {{PROJECT_TAG_EXAMPLES}}.
    모노리포는 `[프로젝트(모듈)]` 처럼 모듈까지 표기.
  - 개발 업무가 아닌 항목(문서 작성, 회의, 리서치 등)은 AI가 성격에 맞는 태그를 임의로 부여할 것.
    예: `[문서]`, `[회의]`, `[리서치]`, `[보고]`. 한번 쓴 태그는 재사용해 일관성 유지.
  - 기록 전 반드시 셸에서 `date` 명령으로 **실제 오늘 날짜를 확인**해 주차 파일을 결정할 것.
    세션이 오래 유지되면 컨텍스트의 날짜는 과거(세션 시작일)일 수 있으므로 신뢰하지 말 것.
  - 해당 주차 파일이 없으면 `{{LOG_PATH}}/_TEMPLATE.md` 구조로 새로 생성하고, 이전 주 미완료 `[ ]` 항목을 이월할 것.
  - 기존 로그는 절대 덮어쓰지 말고 append. 사소한 잡담·단순 질문은 기록하지 말 것.
  - 로그 조회/요약 요청("이번 주 뭐 했어", "저번 주 그 이슈" 등)은 what-did-i-do 스킬을 따를 것.
