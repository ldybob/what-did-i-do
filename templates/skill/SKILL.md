# what-did-i-do

주간 업무 로그 저장소: `{{LOG_PATH}}/weekly_log_YYYY-MM-DD.md`
(파일명 날짜 = 해당 주 **월요일**. 템플릿: `_TEMPLATE.md`)

## 태그 규칙

- 개발 업무: 프로젝트/모듈 경로 태그 — {{PROJECT_TAG_EXAMPLES}},
  모노리포는 `[프로젝트(모듈)]` 처럼 모듈까지 표기
- 비개발 업무: 성격에 맞는 태그를 임의 부여 — `[문서]`, `[회의]`, `[리서치]`, `[보고]` 등.
  새 태그를 만들기 전에 기존 로그에서 쓰던 태그가 있는지 확인하고 재사용

## 동작

> **공통 선행 단계**: 어떤 동작이든 먼저 셸 명령(`date +%F`, PowerShell은 `Get-Date`)으로
> 실제 오늘 날짜를 확인해 이번 주 월요일을 계산한다.
> 컨텍스트의 날짜는 세션 시작일이라 과거일 수 있으므로 신뢰하지 않는다.

### 1. 조회/요약 ("이번 주 뭐 했어?")

1. 실제 오늘 날짜 기준 이번 주 월요일을 계산해 해당 파일을 읽는다
2. 없으면 파일명 정렬 기준 가장 최근 `weekly_log_*.md`를 찾는다
3. 태그별로 묶어 요약. 원문 항목을 그대로 나열하지 말고 흐름으로 정리

### 2. 과거 검색 ("저번 주 그 이슈", "그 프로젝트 언제 했더라?")

1. `{{LOG_PATH}}` 안의 `weekly_log_*.md` 파일들을 키워드/태그로 검색
2. 프로젝트 단위 질문은 태그(`[프로젝트명` 등)로 검색
3. 결과는 주차(파일 날짜)와 함께 시간순으로 제시

### 3. 기록 ("~했어", "로그에 남겨줘")

1. 이번 주 파일이 없으면: `_TEMPLATE.md` 구조로 생성하고, 직전 주차의 미완료 `[ ]` 항목을
   "이번 주 목표"로 이월
2. `## 기록` 섹션에 `- [태그] 내용` 형식으로 **append** (기존 내용 절대 덮어쓰기 금지)
3. 디버깅/설계 여정처럼 맥락이 중요한 항목은 원인·결정·결과가 남게 2~3줄로
4. 인사이트성 내용은 `## 이번 주 배운 것`에 기록

### 4. 보고서 초안 ("주간 보고서 써줘")

1. 이번 주 로그를 읽고 태그(프로젝트)별로 묶어 보고서 초안 작성
2. 완료/진행중/예정 구분, 이슈·리스크는 별도 항목으로

### 5. 정리 ("이번 주 기록 정리해줘", "로그 정리해줘")

**사용자가 명시적으로 요청할 때만 수행한다.** 평소 기록은 상세하게 쌓고, 정리는 이 요청으로 한다.

1. 대상 주차 로그(기본: 이번 주)를 읽고, 같은 주제로 쪼개져 있는 항목들을 찾는다
   (예: "X 검토", "X 1단계", "X 2단계", "X 리뷰 반영" → 전부 주제 X)
2. 주제별로 **하나의 항목으로 통합**: 설계/방향 결론, 구현 완료 범위, 검증 결과, 잔여 작업만 남기고
   중간 시행착오·단계별 진척·세부 수정 과정은 제거
3. 교훈성 내용(실수에서 배운 것, 재발 방지 원칙)은 `## 이번 주 배운 것` 섹션으로 이동
4. 관련 문서 경로, 날짜 범위, "진행 중/완료" 상태는 보존
5. 통합 결과를 사용자에게 요약 보고 (몇 개 항목이 몇 개로 정리됐는지)

## 주의

- 사소한 잡담, 단순 질문·답변은 기록하지 않는다
- 기록 후 git 커밋은 하지 않는다 (사용자가 요청할 때만)
- 파일 인코딩은 UTF-8 유지 (한글 깨짐 방지)
