# UNINSTALL.md — AI 제거 지침

> 이 문서는 사람이 아니라 **AI(Claude Code 등)가 읽고 수행하는 제거 절차**입니다.

## 1. 설치 위치 파악

- 먼저 설정 파일 안의 **"규칙 변경 시 다음 위치에 모두 반영"** 목록을 찾으세요
  (설치 시 2곳 이상에 설치했다면 이 목록이 있습니다). 그 목록이 제거 대상 전체입니다.
- 목록이 없으면 표준 경로를 직접 스캔하세요:
  - `~/.claude/CLAUDE.md` / 윈도우 `C:\Users\<사용자>\.claude\CLAUDE.md`
  - `~/.claude/skills/what-did-i-do/` / 윈도우 `C:\Users\<사용자>\.claude\skills\what-did-i-do\`
  - `~/.gemini/GEMINI.md`
  - `~/.codex/AGENTS.md`

## 2. 규칙 섹션 제거

- 각 설정 파일에서 `# Weekly Log (업무 기록)` 섹션을 제거합니다
  (섹션 헤딩부터 다음 같은 레벨 헤딩 직전까지. GEMINI.md/AGENTS.md는 조회·정리·동기화 규칙 섹션 포함).
- **그 외 내용은 절대 건드리지 마세요.** 사용자의 다른 규칙이 같은 파일에 있습니다.
- 제거 후 파일이 비면, 파일 자체를 지울지 사용자에게 확인하세요.

## 3. 스킬 폴더 삭제

- `~/.claude/skills/what-did-i-do/` 폴더를 삭제합니다 (윈도우 병행 설치면 양쪽 모두).
- 구버전 설치명(`weekly-log`)으로 설치돼 있다면 해당 폴더를 확인 후 삭제합니다.

## 4. 로그 폴더 — 기본은 보존

- **로그 폴더(`weekly_log_*.md`)는 사용자의 업무 기록 데이터입니다. 기본적으로 보존하세요.**
- 사용자가 명시적으로 삭제를 원할 때만, "복구 불가"임을 경고하고 재확인 후 삭제합니다.

## 5. 마무리

1. 제거 결과를 보고합니다: 제거한 위치 목록, 로그 폴더 보존/삭제 여부.
2. 안내: "열려 있는 AI 세션에는 규칙이 남아 있을 수 있으니, 새 세션부터 완전히 반영됩니다."
