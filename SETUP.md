# SETUP.md — AI 설치 지침

> 이 문서는 사람이 아니라 **AI(Claude Code 등)가 읽고 수행하는 설치 절차**입니다.

## 1. 설치 인터뷰

아래 질문을 **한 번에 하나씩** 물어보세요. 답을 받으면 다음 질문으로 넘어갑니다.

### 질문 1 — 로그 저장 경로

"주간 로그를 어디에 저장할까요?"

- 먼저 사용자 환경을 파악하세요 (WSL인지, 네이티브 리눅스/맥인지, 윈도우인지).
- **WSL 사용자이거나 윈도우 도구(윈도우 Claude Code, Antigravity)를 병행할 가능성이 있으면
  윈도우 파일시스템 경로를 추천하세요** (예: `C:\work\weekly-log` = WSL의 `/mnt/c/work/weekly-log`).
  이유: 윈도우 경로는 WSL·윈도우 양쪽에서 항상 접근 가능하지만, WSL 홈은 윈도우에서 불안정합니다.
- 단일 환경(맥, 순수 리눅스)이면 `~/weekly-log`면 충분합니다.

### 질문 2 — 프로젝트 이름 (선택)

"주로 작업하는 프로젝트(레포) 이름들을 알려주세요. 모노리포가 있으면 그것도 알려주세요.
(모르겠거나 나중에 정하고 싶으면 스킵해도 됩니다)"

- **이 질문의 목적**: 규칙에 실제 프로젝트명 태그 예시를 박아 넣어, AI가 세션마다
  태그를 다르게 만드는 문제(`[결제]` vs `[payment]` vs `[payment-api]`)를 방지하고
  나중에 태그 검색이 일관되게 동작하도록 하기 위함입니다.
- 답변으로 태그 예시를 만듭니다: `[프로젝트A]`, `[프로젝트B]`, 모노리포는 `[프로젝트(모듈)]` 형식.
- 스킵하면 `{{PROJECT_TAG_EXAMPLES}}`를 `` `[프로젝트]`(작업 폴더명 기준) ``로 치환하고,
  "AI가 작업 디렉토리 이름에서 태그를 유추하되, 한번 쓴 태그를 재사용"하도록 안내 문구를 넣습니다.

### 질문 3 — 기록 시점/밀도

"업무 기록을 어떤 밀도로 쌓을까요?

1. **상세형 (기본 추천)** — 이벤트가 생길 때마다 즉시 상세 기록. 놓치는 게 없지만 항목이 잘게 쌓임.
   지저분해지면 '이번 주 기록 정리해줘'로 주제 단위 통합.
2. **주제형** — 작업 주제의 결절점(설계 확정, 주제 완료, 배포)에서만 기록. 로그가 깔끔하지만
   중간 과정(시행착오, 단계별 진척)은 남지 않음."

- 선택에 따라 아래 "기록 스타일 반영"대로 규칙 텍스트를 조정합니다.

### 질문 4 — 사용하는 AI 도구

"어떤 AI 도구를 쓰세요? (복수 선택 가능)
1. Claude Code — WSL/리눅스/맥
2. Claude Code — 윈도우
3. Antigravity (Gemini)
4. Codex CLI"

- 답변에 따라 아래 설치 단계에서 해당 도구의 설정만 반영합니다.

## 2. 설치 수행

인터뷰 결과로 `templates/` 안의 파일들의 플레이스홀더를 치환해 설치합니다.

| 플레이스홀더 | 값 |
|---|---|
| `{{LOG_PATH}}` | 질문 1의 경로 (해당 도구 기준 표기로 — WSL 설정엔 `/mnt/c/...`, 윈도우 설정엔 `C:\...`) |
| `{{PROJECT_TAG_EXAMPLES}}` | 질문 2 기반 태그 예시 (스킵 시 위 질문 2의 대체 문구) |
| `{{INSTALLED_LOCATIONS}}` | 질문 4에서 실제 설치한 설정 파일 위치 목록 |

### 기록 스타일 반영 (질문 3)

- **상세형**: 템플릿 그대로 사용 (기본값).
- **주제형**: `claude-rules.md` / `generic-rules.md`의 기록 트리거 부분
  ("대화 중 아래 이벤트가 발생하면 즉시 ..." 항목)을 아래 텍스트로 교체:

  ```
  - 기록 단위는 세부 변경이 아니라 **작업 주제 단위**로 느슨하게. 주제의 결절점에서만 기록할 것:
    방향/설계 확정, 주제 단위 구현 완료, 배포, 굵직한 문제 해결, 사용자가 전달한 회의·일정 내용
  - 진행 중인 주제의 중간 시행착오·단계별 진척·세부 수정·리뷰 반영 같은 것은 기록하지 말 것.
    같은 주제의 항목이 이번 주에 이미 있으면 새 항목을 추가하지 말고 기존 항목을 갱신·통합할 것.
  ```

- 설치 후에도 언제든 바꿀 수 있다고 사용자에게 안내하세요
  ("기록을 더 느슨하게/상세하게 해줘"라고 AI에게 요청하면 규칙 텍스트를 교체 — 설치된 모든 도구 설정에).

### 단계

1. **로그 폴더 생성**: `{{LOG_PATH}}` 생성 후 `templates/_TEMPLATE.md`를 그 안에 복사.
2. **Claude Code (사용 시)**: `templates/claude-rules.md`의 내용(치환·스타일 반영)을
   전역 CLAUDE.md에 **append** (없으면 생성):
   - WSL/리눅스/맥: `~/.claude/CLAUDE.md`
   - 윈도우: `C:\Users\<사용자>\.claude\CLAUDE.md`
   - WSL과 윈도우 둘 다 쓰면 **양쪽 모두** 반영 (경로 표기를 각 환경에 맞게).
3. **what-did-i-do 스킬 설치 (Claude Code 사용 시)**: `templates/skill/SKILL.md`(치환)를
   `~/.claude/skills/what-did-i-do/SKILL.md`로 복사. 윈도우 병행 시 윈도우 쪽 `.claude\skills\`에도.
4. **Antigravity/Gemini (사용 시)**: `templates/generic-rules.md`(치환·스타일 반영)를
   `~/.gemini/GEMINI.md`에 **append** (기존 내용 보존).
5. **Codex CLI (사용 시)**: `templates/generic-rules.md`(치환·스타일 반영)를
   `~/.codex/AGENTS.md`에 **append** (기존 내용 보존, 없으면 생성).
6. **동기화 규칙**: 설치한 곳이 2곳 이상이면, 각 설정 파일의 규칙 끝에
   "규칙 변경 시 다음 위치에 모두 반영: {{INSTALLED_LOCATIONS}}" 항목을 추가.

### 주의사항

- 기존 설정 파일의 내용을 **절대 삭제하거나 덮어쓰지 마세요.** append만 합니다.
- 파일 인코딩은 UTF-8을 유지합니다.

## 2-1. 재설치 / 업데이트 (이미 설치된 경우)

전역 설정에 `# Weekly Log` 섹션이 이미 있으면 재설치(업데이트)로 처리합니다.
**인터뷰를 다시 하지 마세요** — 기존 설정값을 재사용합니다:

1. 기존 설치본에서 값을 추출합니다: 로그 경로, 프로젝트 태그 예시, 기록 스타일(상세형/주제형 —
   "즉시 기록" 문구면 상세형, "주제 단위" 문구면 주제형), 설치 위치 목록(동기화 규칙에서).
2. 추출한 값을 사용자에게 한 줄로 보여주고 "이대로 유지할까요?" 확인합니다
   (바꾸고 싶은 값이 있으면 그것만 다시 묻습니다).
3. **설치 위치 목록의 모든 곳**에서 기존 `# Weekly Log` 섹션을 새 템플릿(값 치환·스타일 반영)으로
   교체하고, 스킬 파일도 새 버전으로 덮어씁니다.
4. 사용자가 직접 수정해둔 커스텀 항목이 기존 섹션에 보이면(템플릿에 없는 줄),
   교체 후에도 유지되도록 새 섹션에 함께 옮깁니다.
5. 로그 파일들은 절대 건드리지 않습니다.

## 3. 마무리

1. 설치 결과를 요약해 보여줍니다 (설치 위치 목록, 로그 경로, 기록 스타일).
2. 안내: "새로 여는 AI 세션부터 적용됩니다. 이 레포 폴더는 이제 삭제해도 됩니다
   (남겨두면 나중에 `git pull` 후 재설치로 업데이트할 수 있습니다).
   바로 시험해보세요 — 아무 세션에서나 '오늘 ~~ 했어'라고 말하면 로그에 기록됩니다."

> **이 레포 안의 파일은 수정하지 마세요.** 설치 상태는 레포가 아니라 전역 설정의
> `# Weekly Log` 섹션 존재 여부로 판단합니다 — 그래야 사용자가 `git pull`로 업데이트를 받을 수 있습니다.
