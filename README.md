# 🗓️ what-did-i-do

> "이번 주 뭐 했지?" — AI가 대신 기억해주는 주간 업무 로그 시스템

당신은 일만 하세요. 기록은 AI가 합니다.

Claude Code, Antigravity(Gemini), Codex 같은 AI 코딩 도구와 대화하며 일하다 보면,
그 대화 속에 이미 업무 기록이 다 들어 있습니다. 이 시스템은 AI가 그걸
**주간 로그 파일로 알아서 적게** 만드는 설정 템플릿입니다.

```
나:  "오늘 결제 모듈 버그 잡았어. 원인은 타임존이었어"
AI:  (weekly_log_2026-07-06.md에 [payment] 항목 자동 기록)

나:  "이번 주 뭐 했지?"
AI:  (로그 읽고 프로젝트별 요약)

나:  "주간 보고서 초안 써줘"
AI:  (완료/진행중/예정으로 정리된 보고서 작성)
```

## 특징

- **여러 AI 도구가 하나의 로그를 공유** — Claude Code(WSL/윈도우), Antigravity(Gemini), Codex CLI가
  같은 폴더에 기록하고 같은 로그를 조회
- **프로젝트 태그** — `[프로젝트]` / `[프로젝트(모듈)]` 태그로 나중에 "그 프로젝트 언제 뭐 했더라" 검색 가능
- **주차 단위 파일** — `weekly_log_YYYY-MM-DD.md` (월요일 날짜), 미완료 항목은 다음 주로 자동 이월
- **날짜 오염 방지** — 세션을 며칠 켜둬도 기록 시점의 실제 날짜로 기록 (`date` 확인 규칙 내장)
- **정리 기능** — 잘게 쌓인 항목들을 요청 시 주제 단위로 통합 ("이번 주 기록 정리해줘")
- **로그는 로컬 전용** — 이 레포는 시스템(규칙+스킬)만 담습니다. 당신의 로그는 어디에도 올라가지 않습니다

## 설치

### 방법 1: AI에게 맡기기 (추천)

이 시스템의 사용자는 어차피 AI 코딩 도구 사용자입니다. 설치도 AI에게 시키세요.

```bash
git clone https://github.com/ldybob/what-did-i-do
cd what-did-i-do
claude   # Claude Code로 열기
```

열고 이렇게 말하면 AI가 설치 인터뷰를 시작합니다:

```
"설치해줘"
"what-did-i-do 설치 시작해줘"
```

(사실 "안녕"이라고만 해도 됩니다 — 레포의 `[SETUP_REQUIRED]` 마커를 보고 AI가 먼저 설치를 제안합니다)

인터뷰는 4개 질문입니다:

1. "로그를 어디에 저장할까요?" (WSL+윈도우 혼용 환경이면 윈도우 경로 추천)
2. "주로 작업하는 프로젝트 이름들을 알려주세요" (태그 일관성용, 스킵 가능)
3. "기록을 어떤 밀도로 쌓을까요?" (상세형 / 주제형)
4. "어떤 AI 도구를 쓰세요?" (Claude Code / Antigravity / Codex, 복수 선택)

답하면 AI가 전역 설정 반영, 스킬 설치, 로그 폴더 생성까지 알아서 끝냅니다.

### 방법 2: 수동 설치

1. 로그 폴더를 만들고 `templates/_TEMPLATE.md`를 복사해 넣기
2. `templates/claude-rules.md`의 내용에서 `{{...}}` 플레이스홀더를 내 값으로 바꿔
   `~/.claude/CLAUDE.md` (전역 CLAUDE.md)에 붙여넣기
3. `templates/skill/SKILL.md`를 `~/.claude/skills/what-did-i-do/SKILL.md`로 복사 (플레이스홀더 치환)
4. (Antigravity 사용 시) `templates/generic-rules.md`를 `~/.gemini/GEMINI.md`에 붙여넣기 (플레이스홀더 치환)
5. (Codex CLI 사용 시) `templates/generic-rules.md`를 `~/.codex/AGENTS.md`에 붙여넣기 (플레이스홀더 치환)
6. AI 도구 세션을 재시작

## 사용법

설치 후에는 그냥 평소처럼 AI와 일하면 됩니다. 아래는 명시적으로 쓸 수 있는 요청들:

| 요청 | 동작 |
|---|---|
| "오늘 ~~ 작업 했어" | 이번 주 로그에 기록 |
| "이번 주 뭐 했어?" | 이번 주 로그 요약 |
| "저번 주 그 이슈 어떻게 됐지?" | 과거 로그 검색 |
| "X 프로젝트에서 언제 했더라?" | 태그 기반 검색 |
| "주간 보고서 초안 써줘" | 보고서 초안 생성 |
| "이번 주 기록 정리해줘" | 쪼개진 항목을 주제 단위로 통합 |

### 태그 규칙

- 개발 업무: 프로젝트/모듈 경로 태그 — `[my-app]`, 모노리포는 `[my-app(auth-module)]`
- 비개발 업무: AI가 알아서 부여 — `[문서]`, `[회의]`, `[리서치]` 등

### 도구별 동작 방식

위 요청들은 도구마다 다른 메커니즘으로 동작합니다 (기능은 동일):

| 도구 | 설치되는 것 |
|---|---|
| Claude Code | 전역 CLAUDE.md 규칙 + `what-did-i-do` 스킬 (`~/.claude/skills/`) |
| Antigravity (Gemini) | `~/.gemini/GEMINI.md`에 규칙 통합 (스킬 시스템 없음) |
| Codex CLI | `~/.codex/AGENTS.md`에 규칙 통합 (스킬 시스템 없음) |

이 레포 자체에는 동작하는 스킬이 없습니다 — `templates/`는 설치용 원본이고,
시스템은 각 도구의 **전역 설정에 설치**됩니다. 설치 후 이 레포 폴더는 삭제해도 됩니다.

## 왜 이렇게 만들었나

- **로그 경로는 환경에 따라 추천이 다릅니다**: 맥이나 리눅스 단일 환경이면 `~/weekly-log`면
  충분합니다. WSL과 윈도우를 오가는 환경이라면 윈도우 경로(`C:\...` = WSL의 `/mnt/c/...`)를
  추천하는데, 윈도우 경로는 양쪽에서 항상 접근 가능하지만 WSL 홈은 윈도우에서
  UNC(`\\wsl$`)로만 접근돼 불안정하기 때문입니다. (설치 인터뷰가 환경을 보고 알아서 추천합니다)
- **기록은 상세하게, 정리는 요청 시에**: 자동으로 느슨하게 기록하면 놓치는 게 생깁니다.
  일단 다 쌓고, 지저분해지면 "정리해줘" 한마디로 주제 단위 통합.
- **기록 전 `date` 확인**: AI가 아는 "오늘"은 세션 시작일이라, 세션을 며칠 켜두면
  주차가 틀어질 수 있습니다. 기록 직전에 실제 날짜를 확인하게 강제합니다.
